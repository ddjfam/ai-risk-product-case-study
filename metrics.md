# Metrics

## Metrics Overview

The success of an AI risk workflow product should be measured using both product and operational metrics. The goal is not only to launch features, but to improve visibility, efficiency, consistency, and stakeholder experience.

## Primary Success Metrics

### 1. Cycle Time
Measures how long it takes for a request to move from intake to final decision.

**Why it matters**
- indicates workflow efficiency
- reveals delays and bottlenecks
- helps track improvement over time

### 2. Request Volume
Measures the number of requests entering the workflow over a given period.

**Why it matters**
- helps assess demand
- supports capacity planning
- identifies growth in workflow adoption

### 3. Completion Rate
Measures the percentage of requests completed within a defined time period.

**Why it matters**
- shows throughput
- helps compare open versus resolved work
- indicates process effectiveness

### 4. Aging Work Items
Measures how many requests remain open beyond expected thresholds.

**Why it matters**
- identifies stalled or blocked work
- highlights priority and ownership issues
- supports operational follow-up

### 5. Intake Quality
Measures the percentage of submissions that are complete and usable on first review.

**Why it matters**
- reflects workflow clarity
- reduces rework
- improves review speed and consistency

## User and Stakeholder Metrics

### 6. Stakeholder Visibility
Measures whether users can easily understand request status, ownership, and next steps.

**Possible indicators**
- reduced manual follow-up
- lower confusion during handoffs
- improved stakeholder feedback

### 7. User Satisfaction
Measures how internal stakeholders rate the usability and usefulness of the workflow experience.

**Possible indicators**
- survey scores
- qualitative feedback
- adoption and repeat use patterns

### 8. Prioritization Clarity
Measures whether stakeholders understand how requests are ranked and why.

**Possible indicators**
- fewer disputes over priority
- clearer decision rationale
- stronger alignment during planning

## Risk and Control-Oriented Metrics

### 9. Decision Traceability
Measures whether decisions are recorded consistently and can be reviewed later.

**Why it matters**
- improves transparency
- supports governance expectations
- reduces reliance on manual tracking

### 10. Workflow Consistency
Measures how reliably teams follow the intended workflow steps.

**Why it matters**
- supports standardization
- reduces process variation
- improves auditability and repeatability

## Example KPI Set

An example KPI set for this product might include:

- average cycle time
- number of open requests
- percent of requests completed on time
- percent of requests submitted with complete intake data
- percent of requests with documented decisions
- user satisfaction score
- number of aging items by workflow stage

## Example KRI Set

An example KRI set might include:

- percent of requests missing required data
- number of requests stalled beyond threshold
- number of requests without assigned owner
- percent of decisions lacking rationale or notes
- repeated bottlenecks at the same workflow stage

## Measurement Principles

Metrics should be:

- simple enough to understand
- useful for decision-making
- aligned to product goals
- consistent across reporting periods
- practical for phased delivery

## Notes

This portfolio example uses generalized metrics to demonstrate product thinking and operational measurement in an enterprise workflow setting.
